# ケアプランAI連携アプリ - 現場テスト（フィールドテスト）実施キット

本資料は、RC版アプリを実際のケアマネジャーに試用（フィールドテスト）してもらうための評価・フィードバック用シートです。印刷して配布するか、Webフォーム等の形式に変換してご利用ください。

---

## 1. テスト実施用 簡易チェックシート（ケアマネジャー向け）

アプリにダミーケース（または匿名化した担当ケース）を入力し、以下のポイントをチェックしてください。

- [ ] **入力の負担感**: 8項目の区分けは多すぎないか。何を書くか迷わないか。
- [ ] **入力順の自然さ**: アセスメントの思考回路（要介護度→現状の課題→環境…）として自然に流れるか。
- [ ] **補助情報機能**: サマリーやメモを丸投げ（テキスト読込）してみて、思った通りに機能するか。
- [ ] **不足情報の役立ち度**: AIによる「アセスメントの不十分な点」の指摘は鋭いか、納得できるか。
- [ ] **注意点の妥当性**: 限度額オーバーの警告や、独居等のリスクに対する警告は実用的か。
- [ ] **出力の利用しやすさ**: 生成された草案文は、介護ソフトにコピペして手直しする土台として使えるか。

## 3. 運営側のテスト見極め＆判断ポイント（次フェーズ向け）

回収したフィードバックを基に、以下の観点で「次フェーズ（Phase 3）の実装優先順位」を決定します。

1. **入力負担がネックになっている場合** -> UI改修（音声入力の追加、項目のさらなる統合、入力内容のローカル保存）を優先。
2. **出力されるサービス案が「現場のリアル」と乖離している場合** -> ローカル知識ファイル（本格RAGやマークダウン管理の拡充）を優先。
3. **「長文メモをもっと賢く読み取ってほしい」という声が強い場合** -> エンジンをGeminiからOpenAI(GPT-4o等)へ切り替える、またはユーザーが選択できるプレミアムUIの実装を優先。
