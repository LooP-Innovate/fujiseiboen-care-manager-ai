# ケアプランAI生成結果 評価基準シート

試験運用において、AIが生成したケアプラン案を専門職（人間）が評価するための基準です。各項目を5段階（または○/×）で評価してください。

## 1. 評価項目

| 評価項目 | 評価ポイント | 評価の観点 |
| :--- | :--- | :--- |
| **妥当性** | 専門職としての適切さ | 介護保険制度やケアマネジメントの原則に則った、妥当な内容か。 |
| **具体性** | 現場での活用可能性 | サービス内容やモニタリング観点が、実務でそのまま通じるほど具体的か。 |
| **過不足** | 情報の精査 | 重要なニーズの漏れはないか。逆に不要な冗長な記述はないか。 |
| **本人意向との整合** | 意思の尊重 | 入力された利用者・家族の希望（デマンド）が長期・短期目標に反映されているか。 |
| **禁止表現の有無** | リスク管理 | 「絶対に」「不明」など、専門職として不適切な表現が含まれていないか。 |
| **修正しやすさ** | 実用性（下書き品質） | 全面書き直しではなく、部分修正で実務に使える状態か。 |

## 2. 総合判定ガイドライン
- **S (極めて良好)**: ほぼ修正なしでそのまま実務（下書き）として利用可能。
- **A (良好)**: 軽微な語尾や表現の修正のみで利用可能。
- **B (可)**: 重要な要素は揃っているが、専門職による一部の書き換えが必要。
- **C (修正が必要)**: 構成は参考になるが、内容の多くを修正する必要がある。
- **D (不適切)**: 誤解や不適切な内容が含まれており、利用できない。

## 3. 運用フロー
1. AIにシナリオを入力し、ケアプラン案を生成。
2. 評価者が本シートに基づき採点。
3. 低評価（C/D）となった項目について、プロンプトの改善案を検討。
